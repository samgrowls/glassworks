## Key Improvements

### 1. Bundled Code Heuristics (Scanner-Level)
**Added to `scanner.rs` and `encrypted_payload_detector.rs`:**
- Skip `/dist/`, `/build/`, `/bin/` directories
- Skip `.min.js`, `.bundle.js`, `.umd.js` files
- Skip files >500KB
- Skip glassware patterns in bundled code
- Skip encrypted_payload in bundled code

**Impact:** 5/10 packages now have 0 findings (were 11-24 findings each)

### 2. Enhanced LLM Prompt
**Added to `analyzer.py`:**
- File type assessment (bundled vs source vs generated)
- Pattern-specific analysis (glassware, encrypted_payload, invisible_char, rc4)
- Author/company assessment (well-known vs anonymous)
- Clear classification criteria with examples
- Explicit FALSE_POSITIVE guidance for bundled/generated code

**Impact:** 3/5 analyzed packages correctly classified as FALSE_POSITIVE
